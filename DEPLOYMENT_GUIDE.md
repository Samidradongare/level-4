# Production Deployment Guide

Follow this guide to deploy the **UsagePay** smart contract, Express backend, and React frontend to production environments.

---

## ⚡ Step 1: Smart Contract Deployment (Stellar Testnet)

### 1. Acquire Testnet XLM
Before deploying, you need an active Stellar testnet keypair funded with gas:
1. Generate a keypair on the [Stellar Laboratory](https://laboratory.stellar.org/#account-creator?network=testnet).
2. Click **Fund with Friendbot** to receive 10,000 free test XLM.
3. Save the public address (starts with `G`) and private seed (starts with `S`).

### 2. Configure Backend Environmental Keys
Open your `backend/.env` file and replace the placeholder credentials:
```env
SERVICE_WALLET_ADDRESS=GD_YOUR_NEW_FRIENDBOT_PUBLIC_KEY
SERVICE_PRIVATE_KEY=SD_YOUR_NEW_FRIENDBOT_SECRET_KEY
```

### 3. Compile and Deploy Contract WASM
1. Navigate to the contract folder and compile:
   ```bash
   cd contract
   cargo build --target wasm32-unknown-unknown --release
   ```
2. Build the deployment scripts and run using npm:
   ```bash
   # Build contract deployment typescript
   npx ts-node ../scripts/deploy-contract.ts
   ```
   **Expected output:**
   ```
   ✅ Contract uploaded!
   ✅ Contract created!
   ✅ Contract deployed successfully!
   Contract Address: CA75UOY4D6I55OETD6Y6ZPXNGL2WCS5WJYYA3R3F5OZHMXL3F4DLUZPP
   ```
3. Update `backend/.env` and `frontend/.env` with the output address:
   ```env
   CONTRACT_ADDRESS=CA75UOY4D6I55OETD6Y6ZPXNGL2WCS5WJYYA3R3F5OZHMXL3F4DLUZPP
   ```

## 🖥️ Step 2: Backend Deployment (Railway or Render)

### Option A: Railway (Recommended)
1. Sign up/Login to [Railway.app](https://railway.app).
2. Click **New Project** -> **Deploy from GitHub Repo** -> Select your backend directory.
3. Add a PostgreSQL plugin inside the Railway dashboard (it automatically links a database URL).
4. Configure environmental variables in the Railway console:
   - `PORT`: `5000`
   - `NODE_ENV`: `production`
   - `JWT_SECRET`: `your_secure_production_key`
   - `CONTRACT_ADDRESS`: `CA75UOY4D6...`
   - `SERVICE_WALLET_ADDRESS`: `GA...`
   - `SERVICE_PRIVATE_KEY`: `SD...`
   - `OPENAI_API_KEY`: `sk-proj-...`
5. Railway compiles TypeScript and runs `npm run build && npm start` automatically.
6. Note the public domain generated by Railway (e.g. `https://usagepay-backend.up.railway.app`).

---

## 🎨 Step 3: Frontend Deployment (Vercel or Netlify)

### Option A: Vercel (Recommended)
1. Download Vercel CLI or login to [Vercel](https://vercel.com).
2. Connect your GitHub repository.
3. Set the root directory of the deployment project to `frontend/`.
4. Configure env variables in the Vercel dashboard:
   - `VITE_API_URL`: Set this to your live backend domain (e.g. `https://usagepay-backend.up.railway.app`).
   - `VITE_CONTRACT_ADDRESS`: Set this to your deployed Soroban contract address.
5. Deploy. The web app should compile and run on HTTPS (e.g. `https://usagepay.vercel.app`).
