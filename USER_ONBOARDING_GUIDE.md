# User Onboarding Strategy Guide

This guide outlines the strategy for recruiting, onboarding, and tracking **10+ real users** generating **50+ transactions** for your Stellar Builder Program submission.

---

## 📈 Onboarding Plan

### Wave 1: Core Friends & Testers (Days 1-2)
- **Goal:** Get 5 active users.
- **Outreach Template:**
  ```text
  Hey! I built a metered payments gateway app on the Stellar testnet for my project submission. I need a few friends to beta test it.
  
  What you get:
  - Free testnet XLM to play with.
  - Test out cool blockchain study-aid tools (summarizes study notes instantly).
  - Help me submit my project!
  
  To help:
  1. Install Freighter wallet browser extension.
  2. Go to: https://usagepay.vercel.app
  3. Click "Connect Wallet" and send me your public G... address so I can fund it.
  4. Pasteurize some raw class notes to generate summaries.
  ```

### Wave 2: Tech Communities & Classrooms (Days 3-4)
- **Goal:** Reach 10+ total users.
- **Outreach channels:** College slack/discord channels, local crypto forums.

---

## 📋 Proof Collection Requirements

For your submission, you must compile screenshots and ledger hashes confirming user interactions:

1. **User Profile Details:** G... wallet address.
2. **Fund Transaction Hash:** Ledger confirmation hash of deposits.
3. **Usage screenshots:** UI screenshots showing decreasing balance and completed summaries.
4. **Stellar Explorer Link:** Clickable link verifying active testnet payment hashes.

### CSV Export format (`USER_PROOF.csv`)
Save active test details in this format:
```csv
wallet_address,user_name,onboarded_date,total_summaries,total_spent_xlm,fund_tx_hash,status
GXXXXXXXXXXXXXXXXXXXX,Friend1,2026-06-25,5,0.05,tx_hash_123...,✅ Completed
```

## 💬 User Feedback Questions
Request your users fill out a quick survey covering these questions:
1. **Onboarding Usability:** How simple was connecting your Freighter wallet? (Scale 1-5)
2. **Billing Transparency:** Did the pay-per-character summary cost feel fair and clear? (Yes/No)
3. **Product Utility:** Would you use a metered blockchain billing system again? (Yes/No)
4. **General Feedback:** What features or improvements would you suggest? (Text)
5. **Net Promoter Score:** Would you recommend this gateway to other creators? (Yes/No)
